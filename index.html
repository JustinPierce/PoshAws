Dependencies

PoshAws needs the AWS SDK assembly to work. You can get it from Amazon or NuGet.

After you have the SDK assembly, you'll need to import it in each PowerShell session before using PoshAws.

  • Import-AwsSdk, when the assembly is in your current directory (or a sub-directory).
  • Import-AwsSdk -SearchPath ..\some-otherdirectory, when it's not.

Credentials

PoshAws needs your AWS credentials to work. You can create them with New-AwsCredentials. The returned credentials can be passed to any PoshAws command with the Credentials parameter.

New-AwsCredentials -AccessKeyId YOUR_ACCESS_KEY_ID -SecretAccessKey YOUR_SECRET_SHHHH

Doing this every time can be a real pain. PoshAws can store your credentials for you. Your credentials are stored in plain text in your PowerShell profile directory; so, be sure you understand how to protect your Windows profile.

New-AwsCredentials -AccessKeyId YOUR_ACCESS_KEY_ID -SecretAccessKey YOUR_SECRET_SHHHH `
  | Add-StoredAwsCredentials -Name WhateverNameYouWouldLike

Or, even simpler.

Add-StoredAwsCredentials -Name WhateverNameYouWouldLike -AccessKeyId YOUR_ACCESS_KEY_ID -SecretAccessKey YOUR_SECRET_SHHHH

You can get your credentials back with Get-StoredAwsCredentials -Name WhateverNameYouUsed and list your stored credentials with Get-StoredAwsCredentialNames.

Providing credentials to every command is tedious. To avoid this, you can use Push-AwsCredentials and Pop-AwsCredentials. For example:

Get-StoredAwsCredentials -Name EasyPeasy | Push-AwsCredentials
# A slew of commands
Pop-AwsCredentials

One last thing. If your commands are running on an EC2 instance with an IAM role, PoshAws will use the role's credentials without you having to do a thing.
